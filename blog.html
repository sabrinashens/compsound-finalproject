Live Coding: Chord-Scale


Dec 14, 2022
Sabrina(Xinyu) Shen


Computer science and music both depend on a system of notations, which is created to facilitate structured communication within its domain.

Thanks to the class Computational Sound, offered in the department of CS at Barnard Collage, we’ve learned to perform and implement with live coding environments. For this project, I aimed to continue this and design a live coding language that employs the collection of musical symbols in the realm of music.



Figure 1: What's in Interface


The Interface has a text area where allows user to enter a space-separated list of musical symbols. Each chord symbol requires the features: Root+Chord+Scale, e.g., Cm9(C minor 9th) or A7(A major 7th). There are some optional features following the structure Beats@Root+Chord+Scale’Inversion[Loops], e.g., 4@Cm9(note length of 4), Cm9’1(first inversion), or Cm9[2](Repeats twice). I will walk you through the details later. Under the text area, there is a slider changing the tempo of the music. User can also select a waveform and at least one envelope(s) of the oscillating signal. Remember to adjust the parameters before clicking “Play”!



As I mentioned, the required features of each unit are Root, Chord, and Scale.

Root is a root key. The availble root keys are C, C#, D, D#, E, F, G, G#, A, A#, B.

Figure 2: Piano Octave

Chord is the chord quality. The available qualities are Major, Minor, Diminished, Augmented.

Figure 3: Chord Symbols

Scale starts at the root and counts up until nth degree.

Figure 4: Scale Symbols


The optional features are Beats, Inversion, and Loops.

Beats is the note length, by entering a positive integer with @. E.g., 2@.

Figure 5: Suggested beats

Inversion is made by moving first n note(s) up by an octave, by entering a positive integer with '. E.g., '2.

Figure 6: Inversion Symbols

Loops is how many time the unit will be played, by entering [n], e.g., [2] means to play twice.

Implementation: Code

If beats inputed

Beats: split by @, the first element of the array

If loops inputed

Loops: split by [, the second element of this array, with "]"removed

If inversion inputed

Inversion: split by ', the second element of this array

Optional features: Beats, Loops, and Inversion are either default or obtained

What left is required: Root, Chord, Scale

The last character of the remaining is the scale number

using a math formula to convert the scale into a number of notes

slice the scale number -> The root key + chord choice

Take the root key and map it onto the pitch

With the root pitch, extend it to the corresponding chord-scale according to its chord

As we get the basic chord-scale, we start doing inversion

Take the number of inversion as x, we take the values of first x element(s), then pitch+12 add to the end

Then slice out the first x elements

What is known now: the inverted or basic chord-scale with loops and beats if given

Find each note the start time and endtime. The first note starts by the start time 0

and the end time is determined by beats/tempo.

diff between beats and tempo: tempo is the same for all notes, beats differ each note's length

The greater beats is, the note plays slower

The greater tempo is, the entire melody plays faster

Then the notelist with note pitch, start time, and end time, which would be ready to play with osc
